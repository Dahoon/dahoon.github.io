Happy New Year 2016

2016-01-01

2016년 목표 리스트를 적었다.

일반 생활

  • 사진 찍기
  • 블로그 하기

언젠가부터 사진을 찍지 않게 되었다. 일본에 온 지 3년이 돼가는데 기억 속 추억 말고는 남은 게 없는 것 같아 아쉬움이 있다. 올해부터는 사진을 부지런히 찍어야겠다.

새로운 언어를 배우면 제일 먼저 하는 게, 웹 프레임워크를 찾아서 블로그를 만든다. 그리고는 그만둔다. 계속 이 패턴을 반복해 왔는데, 올해만큼은 끝까지 블로그를 운영해 볼 생각이다.

프로그래밍

  • Java Script 닌자
  • 사이드 프로젝트 : Elixir, iOS (Swift), Android
  • 1일 1커밋 이상
  • Scala, Haskell, Clojure

올해는 처음으로 프론트 엔드를 주 업무로 하게 돼서, 주력 언어는 Java Script가 될 것 같다. Java Script는 이제까지 진지하게 공부한 적이 없었기 때문에, 깊이 있게 공부해봐야겠다. 우선은 1월 안에 유명한 책들을 한 번씩 읽어볼 참이다.

작년 말부터 공부했던 Elixir로 사이드 프로젝트를 완성할 예정이다. Phoenix 프레임워크도 사실 버그 패치를 보낼 기회가 몇 번이나 있었는데, 이직 준비 때문에 손 놓고 있었던 게 아쉬움이 남는다. 아직은 단순히 버그 패치 수준이지만, 꾸준히 팔로우해서 언젠가는 그럴싸한 풀 리퀘스트를 보내고 싶다.

1일 1커밋이상은 올해는 제대로 한번 도전해봐야겠다. '집에 돌아가서 술 한잔 마시고 자야겠다.' 라는 매일 반복되는 패턴만 없앤다면 불가능해 보이지는 않는다.

배우고 싶은 언어는 Haskell, Scala, Clojure 다. 딱 여기까지만 언어 공부를 하고 그다음부터는 좀 깊이 있게 파야겠다. Java Script나 Elixir도 함수형 프로그래밍 언어이긴 하지만 내가 함수형 프로그래밍에 대해 이론적인 지식이 있는가 하면 그런 것도 아니다. 그래서 스칼라로 함수형 프로그래밍에 대한 이해를 좀 넓힌 다음에 Haskell을 공부하고 싶다. 그리고 Paul Graham의 팬으로서 항상 Lisp를 공부하고 싶다는 생각이 있었는데, Clojure로 Lisp 맛을 좀 볼려고 한다. 올해 한 해는 함수형 프로그래밍에 대한 이해를 확고히 다지는 한 해가 될 것 같다.
